HTML Semantic

Semantic Tags

Semantic HTML introduces meaning to the code we write. Before semantic HTML the elements didn't have any meaning as to what it does or what content goes in. With semantic HTML we were introduced to elements that tell developers and browsers exactly what it does and what content should go in it

Element Placement
Tag Name Description
<header> Header Describes the content at the top of the page <body>. It may include a logo, navigational links or a search bar.
<nav> Navigation Encapsulates the page's navigational links. It is often placed inside the <header> or <footer>
<main> Main content Encapsulates the main content of a page between the header/navigation and the footer areas.
<footer> Footer content Includes the page's footer content at the bottom of the <body>

Embedding media

Semantic HTML introduces us to <video>, <audio> and <embed>

Embedding media
Tag Name Description
<video> Adds video Allows us to add videos to our website. Requires a closing tag.
<audio> Adds audio Allows us to add audio to our website. Requires a closing tag.
<embed> Adds media Used to add any typer of video. It is self-closing tag.

These elements are niversal in that they all use the src attribute to link the source of the content.

Other elements

Semantic HTML introduces us to <figure>, <figcaption>, <section>, <article> and <aside>

Other elements
Tag Name Description
<figure> Adds figure Allows us to add image, diagram or code snippet .
<figcaption> Describe the media Used to describe the media encapsulated within the <figure>.
<section> Defines a doc section Defines elements in the document, such as chapter, headings, or any other documents area with the same theme.
<article> Defines a content Holds content that makes sense on its own such as articles, blogs and comments.
<aside> Additional info Used to mark additional information.
